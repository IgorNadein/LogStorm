# 🔒 Безопасность LogStorm

## Защита секретов

### ✅ Правильно:

1. **Используйте .env файл:**
   ```bash
   # .env (не коммитится в Git)
   GIGACHAT_API_KEY=ваш_настоящий_ключ
   ```

2. **Проверьте .gitignore:**
   ```
   .env
   *.env
   secrets.json
   ```

3. **Используйте .env.example для документации:**
   ```bash
   # .env.example (безопасно коммитить)
   GIGACHAT_API_KEY=your_client_secret_here
   ```

### ❌ Неправильно:

1. **НЕ храните ключи в коде:**
   ```python
   # ❌ ПЛОХО
   api_key = "ваш_настоящий_ключ"
   ```

2. **НЕ коммитьте .env в Git:**
   ```bash
   # ❌ ПЛОХО
   git add .env
   git commit -m "added config"
   ```

3. **НЕ публикуйте ключи в issues/PRs**

## Проверка безопасности

### Перед коммитом:

```bash
# Проверьте, что .env игнорируется
git status

# .env НЕ должен быть в списке изменений
```

### Если случайно закоммитили:

```bash
# 1. Удалите из истории
git rm --cached .env
git commit -m "Remove .env from repository"

# 2. Смените API ключ немедленно!
# Старый ключ скомпрометирован
```

## Получение ключей

### GigaChat API:
1. https://developers.sber.ru/gigachat
2. Войдите через Сбер ID
3. Создайте проект
4. Скопируйте Client Secret

**⚠️ Ключ показывается только один раз при создании!**

## Переменные окружения

### Локальная разработка:
```bash
# .env файл
GIGACHAT_API_KEY=ваш_ключ
```

### Production/Docker:
```bash
# Через переменные окружения
export GIGACHAT_API_KEY="ваш_ключ"

# Или через Docker secrets
docker run -e GIGACHAT_API_KEY="ваш_ключ" logstorm
```

## Утечка данных

### Если ключ скомпрометирован:

1. **Немедленно** смените ключ в панели GigaChat
2. Удалите старый ключ из всех мест
3. Проверьте историю Git на наличие ключа
4. Обновите .env с новым ключом

## Аудит безопасности

Запустите скрипт проверки:

```bash
python setup.py
```

Он проверит:
- ✅ Наличие .env файла
- ✅ Правильность настройки ключей
- ✅ .gitignore конфигурацию
- ✅ Маскировку ключей в выводе

## Best Practices

1. **Разные ключи для разных окружений:**
   - `.env.development` - для разработки
   - `.env.production` - для продакшена

2. **Регулярная ротация ключей:**
   - Меняйте ключи каждые 3-6 месяцев

3. **Минимальные права:**
   - Используйте ключи с минимальными необходимыми правами

4. **Мониторинг:**
   - Следите за использованием API
   - Настройте алерты на подозрительную активность
