# 🎯 Файл профилей - теперь необязательный!

## 📋 Что изменилось в v2.6

Файл `person_prefs.json` теперь **НЕОБЯЗАТЕЛЬНЫЙ**. Система может работать без него!

## ✅ Режимы работы

### 1️⃣ С файлом профилей (как раньше)
```json
{
  "igor_nadein": {
    "display_name": "Игорь Надеин",
    "start_time": "08:00",
    "end_time": "17:00"
  }
}
```
**Результат:**
- ✅ Анализируются только указанные сотрудники
- ✅ Используются индивидуальные графики
- ✅ Отображаются читаемые имена в отчётах

### 2️⃣ БЕЗ файла профилей (НОВОЕ!)
```python
# Просто не указываете файл или удаляете его
prefs = {}
```
**Результат:**
- ✅ Анализируются **ВСЕ** сотрудники из логов
- ✅ Используются дефолтные настройки:
  - `start_time`: 08:00
  - `end_time`: 17:00
  - `workdays`: Пн-Пт
  - `expected_hours`: 9.0
- ⚠️ В отчётах будут системные ID (не читаемые имена)

## 🖥️ Использование в GUI

### Вариант 1: С профилями
1. Запустить `python run_gui.py`
2. Выбрать файл логов
3. Выбрать файл профилей
4. Запустить анализ

### Вариант 2: БЕЗ профилей (быстрый старт)
1. Запустить `python run_gui.py`
2. Выбрать файл логов
3. **Очистить поле "Файл профилей"** или оставить несуществующий путь
4. Система спросит: "Продолжить без профилей?" → Да
5. Запустить анализ

## 🎓 Когда использовать каждый режим?

### ✅ С профилями - для продакшена
**Когда:**
- Регулярный мониторинг посещаемости
- Нужны читаемые отчёты для HR
- Разные графики у сотрудников
- Нужна фильтрация (исключить гостей, посетителей)

**Преимущества:**
- 📊 Красивые отчёты
- 🎯 Точная фильтрация
- ⚙️ Индивидуальные настройки
- 📈 Правильная статистика

### ⚡ БЕЗ профилей - для экспресс-анализа
**Когда:**
- Быстрый одноразовый анализ
- Новая система (профили ещё не готовы)
- Тестирование на новых данных
- Нужно увидеть ВСЕ записи

**Преимущества:**
- 🚀 Быстрый старт (нет настройки)
- 📋 Видны все пользователи
- 🔍 Ничего не упущено
- ⚡ Нет зависимостей

## 💡 Рекомендации

### Для первого запуска:
1. **Запустите БЕЗ профилей** на тестовых данных
2. Посмотрите, какие `name` приходят в логах
3. Создайте `person_prefs.json` на основе этих данных
4. Запустите заново С профилями

### Создание профилей из логов:
```python
# Быстрый скрипт для генерации шаблона
import json
from services import DataLoader

df = DataLoader.load_logs("logs/attendance.csv")
unique_users = df['name'].unique()

prefs = {}
for user in unique_users:
    prefs[user] = {
        "display_name": user.replace("_", " ").title(),
        "start_time": "08:00",
        "end_time": "17:00"
    }

with open("path/person_prefs_template.json", "w") as f:
    json.dump(prefs, f, indent=2, ensure_ascii=False)

print(f"Создан шаблон для {len(prefs)} пользователей")
```
