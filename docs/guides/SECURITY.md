# Безопасность LogStorm

## Секреты

Core/collector контур сейчас не требует `.env`. Доступы к устройствам СКУД задаются в локальном collector-конфиге.

Не коммитьте:

- реальные пароли устройств;
- `.env`;
- локальный `collector.local.py`;
- сетевые пути и учетные данные production-окружения.

## Collector config

Создавайте локальный конфиг из примера:

```bash
python main.py collector --init
```

Проверьте, что файл с реальными паролями игнорируется Git. В репозитории можно хранить только обезличенные примеры вроде `collector/collector.example.py`.

## Перед коммитом

```bash
git status
python -m pytest
```

Проверьте, что в изменениях нет:

- паролей;
- IP/сетевых путей production, если они чувствительны;
- сгенерированных отчетов;
- локальных баз collector.

## Если секрет попал в Git

1. Удалите файл из индекса: `git rm --cached <file>`.
2. Смените пароль или токен в источнике.
3. Проверьте историю и договоритесь о чистке истории, если секрет уже ушел в remote.
