# 🚀 Быстрый экспорт событий СКУД - Инструкция

## Проблема
Экспорт большого периода (например, год) занимает **1-2 часа** из-за медленного ответа устройства (~8 секунд на запрос).

## ✅ Решение 1: Экспорт без изображений (БЫСТРЕЕ!)

Если изображения не нужны, **уберите флаг --pic**:

```bash
python export_acs_events.py \
  --host 192.168.1.101 \
  --user admin \
  --password xZ260616737 \
  --start "2025-01-01T00:00:00" \
  --end "2025-12-19T23:59:59" \
  --out vhod.ndjson \
  --verbose
```

**Ускорение: 20-40%**

## ✅ Решение 2: Увеличить размер страницы

Попробуйте увеличить `--page` до 50 или 100:

```bash
python export_acs_events.py \
  --host 192.168.1.101 \
  --user admin \
  --password xZ260616737 \
  --start "2025-01-01T00:00:00" \
  --end "2025-12-19T23:59:59" \
  --pic \
  --out vhod.ndjson \
  --page 50 \
  --verbose
```

**Ускорение: 30-50%** (если устройство поддерживает)

## ✅ Решение 3: Разделение на периоды (ПАРАЛЛЕЛЬНО!)

Используйте скрипт `export_split.py` для автоматического разделения:

```bash
python export_split.py \
  --host 192.168.1.101 \
  --user admin \
  --password xZ260616737 \
  --start 2025-01-01 \
  --end 2025-12-19 \
  --chunk-days 30 \
  --out vhod_full.ndjson \
  --page 50 \
  --verbose
```

**Преимущества:**
- Автоматическое разделение на периоды по 30 дней
- Можно прервать и продолжить
- Промежуточные файлы можно сохранить

## 🔥 Решение 4: Параллельный экспорт (САМОЕ БЫСТРОЕ!)

Запустите несколько экспортов одновременно в разных терминалах:

**Терминал 1 (январь-март):**
```bash
python export_acs_events.py --password xZ260616737 \
  --start "2025-01-01T00:00:00" --end "2025-03-31T23:59:59" \
  --out q1.ndjson --page 50
```

**Терминал 2 (апрель-июнь):**
```bash
python export_acs_events.py --password xZ260616737 \
  --start "2025-04-01T00:00:00" --end "2025-06-30T23:59:59" \
  --out q2.ndjson --page 50
```

**Терминал 3 (июль-сентябрь):**
```bash
python export_acs_events.py --password xZ260616737 \
  --start "2025-07-01T00:00:00" --end "2025-09-30T23:59:59" \
  --out q3.ndjson --page 50
```

**Терминал 4 (октябрь-декабрь):**
```bash
python export_acs_events.py --password xZ260616737 \
  --start "2025-10-01T00:00:00" --end "2025-12-19T23:59:59" \
  --out q4.ndjson --page 50
```

### Объединение файлов:

**Linux/Mac:**
```bash
cat q1.ndjson q2.ndjson q3.ndjson q4.ndjson > vhod_full.ndjson
```

**Windows (PowerShell):**
```powershell
Get-Content q1.ndjson, q2.ndjson, q3.ndjson, q4.ndjson | Set-Content vhod_full.ndjson
```

**Ускорение: в 4 раза!** (с 2 часов до 30 минут)

## 📊 Сравнение методов

| Метод | Время для года | Сложность | Рекомендуется |

| Стандартный (с --pic) | ~2 часа | Легко | ⭐ |
| Без --pic | ~1.2 часа | Легко | ⭐⭐⭐ |
| Больший --page | ~1 час | Легко | ⭐⭐⭐⭐ |

## 📝 Профилирование

Для анализа производительности добавлен флаг `--verbose`:

```bash
python export_acs_events.py --verbose ...
```

Выводит для каждой итерации:
- Время запроса к API
- Время обработки
- Количество событий

## 🐛 Что делать, если медленно?

1. **Проверьте сеть:**
   ```bash
   ping 192.168.1.101
   ```
   
2. **Убедитесь в стабильном подключении** (не Wi-Fi, а Ethernet)

3. **Попробуйте без --pic** - изображения замедляют

4. **Увеличьте --page** - меньше запросов

5. **Используйте параллельный экспорт** - самый быстрый метод
