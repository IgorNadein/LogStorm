# Памятка: Как работает умная классификация

## 🎯 Цель
Не подставлять сотрудников из-за технических сбоев системы учета.

## 🔍 Логика классификации

### Технический сбой (не вина сотрудника)
Признаки того, что система работает некорректно:

1. **Одно появление за день**
   - Камера зафиксировала только приход ИЛИ только уход
   - Возможно: камера была неисправна, человек прошел мимо зоны обзора
   - Результат: критическая недоработка (0-5ч) → СБОЙ

2. **Ночная активность (22:00-06:00)**
   - Фиксация в нерабочее время
   - Возможно: ложное срабатывание, тестирование системы
   - Результат: аномалия → СБОЙ

3. **Критическая недоработка + аномалия**
   - Работник отработал <6 часов ИЛИ недоработал >3 часов
   - И есть одно появление или ночная активность
   - Результат: вероятный СБОЙ

### Реальная проблема сотрудника
Учитываются ТОЛЬКО если нет признаков технического сбоя:

1. **Опоздание >15 минут**
   - Пришел позже графика на 15+ минут
   - Нет признаков сбоя (2+ появления, дневное время)
   - Результат: ПРОБЛЕМА сотрудника

2. **Недоработка 1-3 часа**
   - Отработал меньше нормы на 1-3 часа
   - Нет критических аномалий
   - Результат: ПРОБЛЕМА сотрудника

3. **Критическая недоработка БЕЗ аномалий**
   - <6 часов работы, но система фиксировала нормально
   - 2+ появления, дневное время
   - Результат: ПРОБЛЕМА сотрудника

## 🎨 Цветовая схема Excel

| Цвет | Значение | Действие |
|------|----------|----------|
| 🔴 Красный фон | Технический сбой | Передать в IT для диагностики |
| 🟡 Желтая ячейка | Проблема сотрудника | Разобраться с сотрудником |
| 🟢 Зеленый | Переработка | Похвалить/компенсировать |

## 💡 Как использовать

### Для HR-специалиста
1. Открыть Excel отчет
2. Смотреть колонку **"Проблемы сотрудника"**
3. Игнорировать красные строки (это технические сбои)
4. Работать только с желтыми ячейками

### Для IT-администратора
1. Открыть лист **"Подозрительные случаи"**
2. Анализировать колонку **"Технический сбой"**
3. Проверить камеры в указанные даты
4. Устранить причины сбоев

### Для руководителя
1. Читать раздел **"РЕАЛЬНЫЕ ОПОЗДАНИЯ СОТРУДНИКОВ"**
2. Смотреть процент от **валидных записей**
3. Игнорировать технические сбои в принятии решений
4. Использовать AI-анализ для общих выводов


## ❓ FAQ

**Q: Почему сотрудник с 1 появлением считается сбоем?**
A: Камера должна фиксировать приход И уход. Если есть только одно - система работала некорректно.

**Q: Можно ли считать ночную активность реальной?**
A: В офисах с ночными сменами нужно скорректировать логику в коде (убрать проверку ночного времени).

**Q: Как быть с опозданиями <15 минут?**
A: Они фиксируются в колонке "Опоздание", но не попадают в "Проблемы сотрудника" (считаются допустимыми).
