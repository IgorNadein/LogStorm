# Быстрый старт: Работа с NDJSON логами СКУД

## 🎯 Что нового

Система LogStorm теперь поддерживает:
- ✅ Загрузку логов из системы СКУД в формате NDJSON
- ✅ Изменение имён сотрудников
- ✅ Индивидуальные расписания для каждого
- ✅ Объединение нескольких ID в одного человека (aliases)

## 🚀 Быстрый запуск

### 1. Поместите NDJSON файлы в папку `LogsCam/`

```
LogStorm/
└── LogsCam/
    ├── vhod.ndjson    # Логи входов
    ├── vihod.ndjson   # Логи выходов
    └── events.ndjson  # Все события
```

### 2. Запустите тестовый скрипт

```bash
python test_ndjson_with_mapper.py
```

Скрипт автоматически:
- Загрузит конфигурацию маппинга
- Прочитает логи
- Проанализирует посещаемость
- Создаст Excel отчёт `attendance_report_ndjson.xlsx`

## ⚙️ Настройка маппинга (опционально)

Файл `person_mapping.json` позволяет настроить:

### Изменить отображаемое имя

```json
{
  "person_mappings": {
    "30": {
      "display_name": "Ирина Погонина",
      "original_names": ["Ирина Погонина"]
    }
  }
}
```

### Установить индивидуальное расписание

```json
{
  "person_mappings": {
    "3": {
      "display_name": "Эдуард Employee Sample",
      "workdays": ["Monday", "Tuesday", "Wednesday", "Thursday"],
      "start_time": "10:00",
      "end_time": "19:00",
      "work_hours": 9
    }
  }
}
```

### Объединить несколько ID в одного человека

```json
{
  "person_mappings": {
    "54": {
      "display_name": "Сергей Бондарь",
      "original_names": ["Серёга Бондарь", "Сергей Бондарь"]
    }
  },
  "aliases": {
    "54": ["sergei_bondar_2", "bondar_old_id"]
  }
}
```

Теперь все записи с ID `54`, `sergei_bondar_2` и `bondar_old_id` будут считаться как записи одного человека.

## 📝 Примеры использования

### Анализ одного файла

```python
from services import DataLoader, PersonMapper, AttendanceService

# С маппингом
mapper = PersonMapper('person_mapping.json')
df = DataLoader.load_logs('LogsCam/vhod.ndjson', 
                         file_type='ndjson',
                         person_mapper=mapper)

# Без маппинга (использует имена из логов)
df = DataLoader.load_logs('LogsCam/vhod.ndjson', file_type='ndjson')
```

### Анализ нескольких файлов

```python
files = ['LogsCam/vhod.ndjson', 'LogsCam/vihod.ndjson']
df = DataLoader.load_logs(files, file_type='ndjson', person_mapper=mapper)
```

### Добавить сотрудника программно

```python
mapper = PersonMapper('person_mapping.json')

mapper.add_person(
    person_id='99',
    display_name='Новый Сотрудник',
    workdays=['Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday'],
    start_time='09:00',
    end_time='18:00',
    work_hours=9
)

mapper.save_mappings()
```

## ❓ Частые вопросы

**Q: Как узнать ID сотрудника?**  
A: Откройте NDJSON файл и найдите поле `employeeNoString` в событии сотрудника.

**Q: Можно ли работать без файла `person_mapping.json`?**  
A: Да! Система автоматически создаст дефолтные профили используя имена из логов.

**Q: Как объединить старый и новый ID сотрудника?**  
A: Используйте `aliases`. Укажите актуальный ID как главный, старый добавьте в список aliases.

**Q: Поддерживается ли CSV?**  
A: Да, система работает и с CSV и с NDJSON файлами. Формат определяется автоматически.

## 🆘 Поддержка

При проблемах проверьте:
1. ✅ Формат NDJSON файла корректный (каждая строка = валидный JSON)
2. ✅ В событиях есть поле `employeeNoString` или `name`
3. ✅ Файл `person_mapping.json` валидный JSON
4. ✅ ID в маппинге совпадают с ID из логов
