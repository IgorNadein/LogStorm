# 📥 Экспорт событий из СКУД Hikvision/HiWatch

## 🎯 Назначение

Инструменты для экспорта событий доступа из устройств Hikvision/HiWatch СКУД через ISAPI в формат NDJSON.

## 📋 Файлы

- **export_acs_events.py** - Основной скрипт экспорта событий
- **test_performance.py** - Тесты производительности экспорта
- **EXPORT_GUIDE.md** - Эта инструкция

## 🚀 Быстрый старт

### Базовый экспорт (без изображений)

```bash
python tools/export/export_acs_events.py \
  --host 192.168.1.101 \
  --user admin \
  --password your_password \
  --start "2025-01-01T00:00:00" \
  --end "2025-12-31T23:59:59" \
  --out data/events.ndjson \
  --verbose
```

### Экспорт с изображениями

```bash
python tools/export/export_acs_events.py \
  --host 192.168.1.101 \
  --user admin \
  --password your_password \
  --start "2025-01-01T00:00:00" \
  --end "2025-12-31T23:59:59" \
  --pic \
  --out data/events_with_pic.ndjson \
  --verbose
```

## ⚡ Оптимизация производительности

### Проблема
Экспорт большого периода (год) может занимать **1-2 часа** из-за медленного ответа устройства.

### Решение 1: Без изображений (быстрее на 20-40%)
Уберите флаг `--pic` если изображения не нужны.

### Решение 2: Увеличить размер страницы (быстрее на 30-50%)
```bash
--page 50  # или --page 100
```

Попробуйте увеличить размер страницы, если устройство поддерживает.

### Решение 3: Разделение на периоды
Разбейте большой период на несколько меньших и запускайте параллельно:

```bash
# Январь-Июнь
python tools/export/export_acs_events.py ... --start "2025-01-01" --end "2025-06-30"

# Июль-Декабрь
python tools/export/export_acs_events.py ... --start "2025-07-01" --end "2025-12-31"
```

## 📊 Параметры

### Обязательные
- `--host` - IP адрес устройства
- `--user` - Имя пользователя
- `--password` - Пароль
- `--start` - Дата начала (формат: `YYYY-MM-DDTHH:MM:SS`)
- `--end` - Дата окончания (формат: `YYYY-MM-DDTHH:MM:SS`)

### Опциональные
- `--out` - Файл вывода (по умолчанию: `events.ndjson`)
- `--pic` - Включить экспорт изображений (base64)
- `--page` - Размер страницы (по умолчанию: 20, макс: 100)
- `--verbose` - Подробный вывод
- `--timeout` - Таймаут запроса в секундах (по умолчанию: 30)

## 📄 Формат вывода (NDJSON)

Каждая строка - JSON объект события:

```json
{"major": 5, "minor": 75, "time": "2025-12-01T06:05:04+03:00", "cardNo": "12345", "name": "Иван Иванов", "employeeNoString": "55", "serialNo": 97659}
{"major": 5, "minor": 104, "time": "2025-12-01T17:30:15+03:00", "cardNo": "12345", "name": "Иван Иванов", "employeeNoString": "55", "serialNo": 97662}
```

### Коды событий (minor)
- **75** - Успешный вход (карта/лицо)
- **104** - Успешный выход (карта/лицо)
- **21** - Дверь открыта
- **22** - Дверь закрыта
- **76** - Неопознанное лицо/карта

## ⚠️ Возможные проблемы

### 1. Таймаут соединения
**Проблема:** Устройство не отвечает.
**Решение:** Увеличьте `--timeout 60` или проверьте доступность устройства.

### 2. Ошибка 401 Unauthorized
**Проблема:** Неверные логин/пароль.
**Решение:** Проверьте учётные данные.

### 3. Медленный экспорт
**Проблема:** Экспорт занимает слишком много времени.
**Решение:** 
- Уберите `--pic` если изображения не нужны
- Увеличьте `--page` до 50-100
- Разделите период на части

### 4. Недопустимый размер страницы
**Проблема:** `Invalid pageSize`.
**Решение:** Уменьшите `--page` до 20-30.
