## [v2.0] - Умный анализ с разделением технических сбоев

### 🎯 Основное улучшение
Теперь система **различает технические сбои и реальные проблемы сотрудников**, что предотвращает несправедливые обвинения из-за багов системы.

### 🔍 Что изменилось

#### 1. Новые колонки в Excel
- **"Технический сбой"** - возможные сбои системы (одно появление, ночная активность, критические аномалии)
- **"Проблемы сотрудника"** - реальные нарушения (опоздания >15 мин, недоработка <3ч)

#### 2. Умная классификация проблем

**Технические сбои (НЕ вина сотрудника):**
- Одно появление за день (возможно, камера не зафиксировала все проходы)
- Ночная активность 22:00-06:00 (аномалия системы)
- Критическая недоработка + аномалия = вероятный сбой

**Реальные проблемы сотрудника:**
- Опоздание более 15 минут (без признаков сбоя)
- Недоработка 1-3 часа (без признаков сбоя)
- Систематические нарушения

#### 3. Улучшенная статистика

**Консольный отчет теперь показывает:**
```
📊 ОБЩАЯ СТАТИСТИКА:
  • Всего записей: 1215
  • Валидных записей: 1009 (83.0%)
  • Технических сбоев системы: 206
  
⏰ РЕАЛЬНЫЕ ОПОЗДАНИЯ СОТРУДНИКОВ:
  • Всего случаев: 274 (27.2% от валидных)
```

**AI анализ получает очищенные данные:**
- Исключены технические сбои из статистики сотрудников
- Учитываются только реальные проблемы дисциплины
- Добавлена оценка качества работы системы учета

#### 4. Улучшенное форматирование Excel

**Цветовая схема:**
- 🔴 **Красный фон** - технические сбои (весь ряд)
- 🟡 **Желтая ячейка** - проблемы сотрудника
- 🟠 **Оранжевый** - опоздание
- 🟡 **Желтый** - недоработка
- 🟢 **Зеленый** - переработка (позитив)

**Лист "Подозрительные случаи":**
Содержит только технические аномалии для анализа IT-специалистами

### 📊 Пример работы

**Старая версия:**
```
Серёга Немакин: 17 опозданий (включая технические сбои)
```

**Новая версия:**
```
Серёга Немакин: 13 реальных опозданий
+ 4 случая технических сбоев (не учитываются)
```

### 💡 Преимущества

1. **Справедливость** - сотрудники не отвечают за баги системы
2. **Точность** - статистика основана на валидных данных
3. **Диагностика** - видно качество работы системы учета
4. **Прозрачность** - AI анализ учитывает только реальные проблемы

### 🔧 Техническая реализация

**Изменения в коде:**
```python
# Разделение проблем
suspicious = []  # Технические сбои
employee_issues = []  # Реальные проблемы

# Логика классификации
if appearances == 1 or night_activity:
    suspicious.append("Возможный сбой")
else:
    if late and late_minutes > 15:
        employee_issues.append("Опоздание")
```

**Фильтрация в статистике:**
```python
# Только валидные данные
df_valid = df_report[df_report['Технический сбой'] == 'Нет']
late_count = len(df_valid[df_valid['Опоздание'] == 'Да'])
```
