# Настройка пороговых значений и проверок

Все пороговые значения для проверок находятся в файле **`config.py`**.

## 📊 Основные настройки

### Рабочее время
```python
OVERTIME_THRESHOLD = 10  # Переработка: более N часов в день
```

### Опоздания
```python
LATE_THRESHOLD_MINUTES = 15  # Опоздание учитывается если больше N минут
CRITICAL_LATE_MINUTES = 180  # Критическое опоздание (3+ часа) = технический сбой
```

### Недоработки
```python
CRITICAL_UNDERWORK_HOURS = 7  # Критическая недоработка: менее N часов
CRITICAL_UNDERWORK_DIFF = 2  # Критическая недоработка: на N+ часов меньше нормы
```

### Массовые отсутствия
```python
MASS_ABSENCE_THRESHOLD = 0.9  # Массовое отсутствие: >90% отсутствуют в рабочий день
CRITICAL_ABSENCE_THRESHOLD = 1.0  # Критическое отсутствие: 100% отсутствуют
```

### Ночное время (для определения аномалий)
```python
NIGHT_HOUR_START = 23  # Начало ночного времени (час)
NIGHT_HOUR_END = 3  # Конец ночного времени (час)
```

## 🔍 Логика проверок

### Технические сбои (красный цвет)
**Применяются всегда, независимо от дня недели:**

1. **Критическое отсутствие (100%)** - все сотрудники отсутствуют
2. **Массовое отсутствие (>90%)** - применяется только для **рабочих дней**
3. **Ночная активность** - приход/уход в 23:00-03:00, только для **рабочих дней**
4. **Одно появление** - камера зафиксировала только приход ИЛИ уход, только для **рабочих дней**
5. **Экстремальное опоздание (>3 часа)** - проверяется автоматически только для рабочих дней (т.к. опоздание не считается для выходных)
6. **Критическая недоработка (<7ч или на 2+ч меньше)** - только для **рабочих дней**

### Проблемы сотрудников (желтый цвет)
**Применяются только для рабочих дней:**

1. **Опоздание** - приход позже графика на >15 минут
2. **Ранний уход** - уход раньше графика
3. **Недоработка часов** - отработано меньше положенного

### Переработки (зеленый цвет)
**Для рабочих дней:** отработано >10 часов  
**Для выходных дней:** любая работа считается переработкой

## 🎨 Цветовое кодирование в Excel

### Главный лист (детализация)
- **Красный** - технические сбои (всегда)
- **Желтый** - опоздания/недоработки (только рабочие дни)
- **Зеленый** - переработки (рабочие дни >10ч)

### Месячный лист (сводка)
- **Красный** - технические сбои (всегда)
- **Желтый** - опоздания/недоработки (только рабочие дни)
- **Зеленый** - переработки (рабочие дни >10ч + **любая работа в выходные**)

## ⚙️ Как изменить настройки

1. Откройте файл **`config.py`**
2. Найдите нужную константу (например, `OVERTIME_THRESHOLD`)
3. Измените значение
4. Сохраните файл
5. Перезапустите анализ

## 📝 Примеры изменений

### Увеличить порог переработки до 11 часов
```python
OVERTIME_THRESHOLD = 11  # Было: 10
```

### Сделать опоздания более строгими (>5 минут)
```python
LATE_THRESHOLD_MINUTES = 5  # Было: 15
```

### Изменить порог критической недоработки
```python
CRITICAL_UNDERWORK_HOURS = 6  # Было: 7
CRITICAL_UNDERWORK_DIFF = 3  # Было: 2
```

## 🔧 Где используются настройки

| Модуль | Используемые константы |
|--------|------------------------|
| `validators/time_validator.py` | CRITICAL_LATE_MINUTES, CRITICAL_UNDERWORK_HOURS, CRITICAL_UNDERWORK_DIFF, NIGHT_HOUR_START, NIGHT_HOUR_END |
| `analyzers/status_analyzer.py` | LATE_THRESHOLD_MINUTES, OVERTIME_THRESHOLD |
| `analyzers/technical_analyzer.py` | (использует методы из validators) |

| `reporters/excel_formatter.py` | (использует цветовые константы) |

## ⚠️ Важные примечания

1. **Рабочие дни определяются персонально** для каждого сотрудника в `person_mapping.json` (поле `workdays`)
2. **Технические сбои имеют наивысший приоритет** - если есть технический сбой, ячейка будет красной независимо от других статусов
3. **Для выходных дней** не применяются проверки опозданий, недоработок и большинство технических сбоев (кроме критического отсутствия)
4. **Любая работа в выходной день** считается переработкой и подсвечивается зеленым
