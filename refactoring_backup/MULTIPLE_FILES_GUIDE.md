# 📚 Множественный выбор файлов - новая возможность!

## 🎯 Что нового в v2.7

Теперь можно **выбрать несколько файлов логов** для одновременного анализа!

## ✨ Возможности

### 1️⃣ Объединение данных из разных источников
```
Файл 1: logs/october.csv      (1000 записей)
Файл 2: logs/november.csv     (1200 записей)
Файл 3: LogsCam/december.ndjson (800 записей)
────────────────────────────────────────────
Итого:  3000 записей в одном отчёте
```

### 2️⃣ Смешивание форматов
- ✅ CSV + CSV
- ✅ NDJSON + NDJSON
- ✅ CSV + NDJSON (смешанный режим!)

### 3️⃣ Автоматическое объединение
- Данные сортируются по времени
- Дубликаты сохраняются (если нужны)
- Единый DataFrame для анализа

## 🖥️ Использование в GUI

### Добавление файлов:
1. Запустите `python run_gui.py`
2. Нажмите **"➕ Добавить файл"**
3. Выберите первый файл → Добавлен в список
4. Нажмите **"➕ Добавить файл"** снова
5. Выберите второй файл → Добавлен в список
6. Повторите для всех нужных файлов
7. Запустите анализ

### Очистка списка:
- Нажмите **"🗑️ Очистить всё"**
- Список файлов будет очищен

### Просмотр списка:
- Все выбранные файлы показаны в списке
- Можно скроллить, если файлов много

## 💻 Использование в CLI

# Одиночный файл (как раньше)

## 🔧 Технические детали

### DataLoader._load_multiple_files()
```python
def _load_multiple_files(paths: list, file_type: str = 'auto'):
    print(f"📚 Загрузка {len(paths)} файлов...")
    
    dfs = []
    for i, path in enumerate(paths, 1):
        print(f"  [{i}/{len(paths)}] {basename(path)}...", end=" ")
        try:
            df = DataLoader._load_single_file(path, file_type)
            dfs.append(df)
            print(f"✅ {len(df)} записей")
        except Exception as e:
            print(f"❌ Ошибка: {e}")
            continue
    
    # Объединяем
    combined_df = pd.concat(dfs, ignore_index=True)
    combined_df = combined_df.sort_values('timestamp')
    
    return combined_df
```

### Обработка ошибок
```
Если один из файлов не загрузится:
- ⚠️ Выводится предупреждение
- ✅ Остальные файлы обрабатываются
- 📊 Анализ продолжается с доступными данными
```

## ⚙️ Настройки

### Автоопределение формата
```python
# Если file_type='auto' (по умолчанию):
# - Каждый файл определяется отдельно
# - october.csv → CSV
# - events.ndjson → NDJSON
# - Смешивание форматов OK!
```

### Принудительный формат
```python
# Если file_type='csv':
# - ВСЕ файлы обрабатываются как CSV
# - Используйте если расширение неправильное
```

## ⚠️ Важные моменты

### 1. Дубликаты НЕ удаляются
```
Если в разных файлах есть одинаковые записи:
- Все сохраняются
- Сортируются по timestamp
- Показываются в отчёте

💡 Если нужна дедупликация - сделайте вручную после
```

### 2. Память
```
Все файлы загружаются в память:
- Маленькие файлы (<100MB) - OK
- Большие файлы (>500MB) - может быть медленно
- Очень большие (>1GB) - лучше объединить заранее
```

### 3. Производительность
```
Загрузка:  линейная O(n)
Объединение: O(n) 
Сортировка: O(n log n)

Итого: ~30 секунд на 100k записей
```

## 💡 Рекомендации

### ✅ Лучшие практики:

1. **Именование файлов**
   ```
   Хорошо:  october_2025.csv, november_2025.csv
   Плохо:   log1.csv, log2.csv
   ```

2. **Порядок добавления**
   ```
   - Добавляйте в хронологическом порядке
   - Система всё равно отсортирует, но так нагляднее
   ```

3. **Проверка перед анализом**
   ```
   - Посмотрите список в GUI
   - Убедитесь что все файлы добавлены
   - Проверьте пути
   ```

### ⚠️ Чего избегать:

1. **Слишком много файлов**
   ```
   10-20 файлов: OK
   50+ файлов: Медленно
   100+ файлов: Лучше объединить заранее
   ```

2. **Огромные файлы**
   ```
   <100 MB: Отлично
   100-500 MB: Нормально
   >500 MB: Подумайте о разбиении
   ```
