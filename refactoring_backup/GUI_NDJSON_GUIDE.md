# LogStorm GUI - Работа с NDJSON файлами

## Обзор

LogStorm GUI теперь полностью поддерживает NDJSON файлы от систем СКУД Hikvision/HiWatch с использованием PersonMapper для маппинга сотрудников.

## Быстрый старт

### 1. Подготовка файлов

Убедитесь, что у вас есть:
- **NDJSON файлы** с событиями СКУД (например, `vhod.ndjson`, `vihod.ndjson`)
- **person_mapping.json** - файл маппинга сотрудников (опционально)

### 2. Запуск GUI

```bash
python gui_app.py
# или
python run_gui.py
```

### 3. Настройка в интерфейсе

#### Вкладка "⚙️ Настройки"

1. **📁 Файлы логов**
   - Нажмите "➕ Добавить файл"
   - Выберите NDJSON файлы (можно выбрать несколько: `vhod.ndjson`, `vihod.ndjson`)
   - Убедитесь, что тип файла установлен на "NDJSON" (или "Авто")

2. **👤 Файл профилей сотрудников** (необязательно)
   - Можно оставить пустым для NDJSON
   - Или указать `person_prefs.json` для совместимости

3. **🔄 Файл маппинга сотрудников** (для NDJSON)
   - Нажмите "Обзор..." и выберите `person_mapping.json`
   - Или оставьте значение по умолчанию
   - Этот файл используется для:
     - Изменения отображаемых имён
     - Объединения нескольких ID в одного сотрудника (алиасы)
     - Установки индивидуальных расписаний
     - Связывания нескольких вариантов имён

4. **💾 Файл отчёта Excel**
   - Укажите путь для сохранения отчёта
   - По умолчанию: `reports/attendance.xlsx`

5. **🔧 Дополнительные опции**
   - ✅ Включить AI анализ (если настроен GigaChat)

6. **▶️ Запустить анализ**
   - Нажмите кнопку для старта

### 4. Просмотр результатов

После запуска переключитесь на вкладку "📊 Анализ" для просмотра:
- Загрузка маппинга (количество сотрудников и алиасов)
- Прогресс обработки файлов
- Статистика анализа
- Генерация отчётов

## Что делает PersonMapper?

### До маппинга (исходные NDJSON данные):
```json
{"name": "Серёга Бондарь", "employeeNoString": "54", "time": "2025-12-16T18:04:40+03:00"}
{"name": "Серёга Бондарь", "employeeNoString": "54", "time": "2025-12-17T08:50:44+03:00"}
```

### После маппинга (person_mapping.json):
```json
{
  "person_mappings": {
    "54": {
      "display_name": "Сергей Бондарь",
      "original_names": ["Серёга Бондарь", "Сергей Бондарь"],
      "workdays": ["Monday", "Tuesday", "Wednesday", "Thursday", "Friday"],
      "start_time": "09:00",
      "end_time": "18:00",
      "work_hours": 9
    }
  }
}
```

**Результат:**
- Имя в отчёте: **"Сергей Бондарь"** (вместо "Серёга Бондарь")
- Расписание: Пн-Пт, 09:00-18:00, 9 часов
- Все события с ID "54" приписаны этому сотруднику

## Работа с алиасами

Если у сотрудника несколько карт/ID:

```json
{
  "person_mappings": {
    "54": {
      "display_name": "Сергей Бондарь",
      "original_names": ["Серёга Бондарь"],
      "workdays": ["Monday", "Tuesday", "Wednesday", "Thursday", "Friday"],
      "start_time": "09:00",
      "end_time": "18:00",
      "work_hours": 9
    }
  },
  "aliases": {
    "54": ["sergei_bondar_2", "bondar_sergey"]
  }
}
```

Теперь события с ID `54`, `sergei_bondar_2` и `bondar_sergey` будут объединены под одним сотрудником **"Сергей Бондарь"**.

## Индивидуальные расписания

Каждому сотруднику можно задать своё расписание:

```json
{
  "person_mappings": {
    "30": {
      "display_name": "Ирина Погонина",
      "original_names": ["Ирина Погонина"],
      "workdays": ["Monday", "Tuesday", "Wednesday", "Thursday"],
      "start_time": "10:00",
      "end_time": "17:00",
      "work_hours": 7
    },
    "54": {
      "display_name": "Сергей Бондарь",
      "original_names": ["Серёга Бондарь"],
      "workdays": ["Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday"],
      "start_time": "08:00",
      "end_time": "20:00",
      "work_hours": 12
    }
  }
}
```

## Преимущества использования маппинга

1. **Нормализация имён**: "Серёга" → "Сергей", "Настя" → "Анастасия"
2. **Объединение дубликатов**: Несколько карт → Один сотрудник
3. **Индивидуальные графики**: Разные расписания для разных людей
4. **Гибкость**: Легко добавлять/изменять сотрудников без изменения исходных логов
5. **Совместимость**: Маппинг конвертируется в формат person_prefs.json автоматически

## Управление маппингом

Для редактирования `person_mapping.json` используйте:

```bash
python manage_mapping.py
```

Интерактивное меню позволяет:
- Просмотреть всех сотрудников
- Добавить/изменить сотрудника
- Управлять алиасами
- Импортировать из NDJSON
- Экспортировать в person_prefs.json

## Troubleshooting

### Проблема: "Файл маппинга не найден"
**Решение:** Создайте `person_mapping.json` с помощью `manage_mapping.py` или используйте GUI без маппинга (дефолтные настройки)

### Проблема: "Невалидные данные на выходе"
**Решение:** 
1. Проверьте, что файл маппинга указан в GUI
2. Убедитесь, что employeeNoString в NDJSON соответствуют ключам в person_mappings
3. Проверьте формат JSON с помощью `python -m json.tool person_mapping.json`

### Проблема: "Сотрудник не найден в маппинге"
**Решение:** 
- GUI будет использовать оригинальное имя из NDJSON
- Добавьте сотрудника в маппинг с помощью `manage_mapping.py`
