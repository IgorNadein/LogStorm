## [v2.0] - Умный анализ с разделением технических сбоев

### 🎯 Основное улучшение
Теперь система **различает технические сбои и реальные проблемы сотрудников**, что предотвращает несправедливые обвинения из-за багов системы.

### 🔍 Что изменилось

#### 1. Новые колонки в Excel
- **"Технический сбой"** - возможные сбои системы (одно появление, ночная активность, критические аномалии)
- **"Проблемы сотрудника"** - реальные нарушения (опоздания >15 мин, недоработка <3ч)

#### 2. Умная классификация проблем

**Технические сбои (НЕ вина сотрудника):**
- Одно появление за день (возможно, камера не зафиксировала все проходы)
- Ночная активность 22:00-06:00 (аномалия системы)
- Критическая недоработка + аномалия = вероятный сбой

**Реальные проблемы сотрудника:**
- Опоздание более 15 минут (без признаков сбоя)
- Недоработка 1-3 часа (без признаков сбоя)
- Систематические нарушения

#### 3. Улучшенная статистика

**Консольный отчет теперь показывает:**
```
📊 ОБЩАЯ СТАТИСТИКА:
  • Всего записей: 1215
  • Валидных записей: 1009 (83.0%)
  • Технических сбоев системы: 206
  
⏰ РЕАЛЬНЫЕ ОПОЗДАНИЯ СОТРУДНИКОВ:
  • Всего случаев: 274 (27.2% от валидных)
```

**AI анализ получает очищенные данные:**
- Исключены технические сбои из статистики сотрудников
- Учитываются только реальные проблемы дисциплины
- Добавлена оценка качества работы системы учета

#### 4. Улучшенное форматирование Excel

**Цветовая схема:**
- 🔴 **Красный фон** - технические сбои (весь ряд)
- 🟡 **Желтая ячейка** - проблемы сотрудника
- 🟠 **Оранжевый** - опоздание
- 🟡 **Желтый** - недоработка
- 🟢 **Зеленый** - переработка (позитив)

**Лист "Подозрительные случаи":**
Содержит только технические аномалии для анализа IT-специалистами

### 📊 Пример работы

**Старая версия:**

4. **Прозрачность** - AI анализ учитывает только реальные проблемы

# Changelog v2.8 - PersonMapper для СКУД

## 🎉 Новые возможности

### PersonMapper - Система маппинга сотрудников из СКУД

Добавлена полноценная система для работы с логами систем контроля доступа (СКУД) в формате NDJSON.

#### Основные возможности:

1. **Маппинг имён сотрудников** 
   - Изменение отображаемых имён в отчётах
   - Поддержка нескольких вариантов написания имени из СКУД
   - Автоматическое сопоставление по ID или имени

2. **Индивидуальные расписания**
   - Настройка рабочих дней для каждого сотрудника
   - Индивидуальное время начала и окончания работы
   - Различное количество рабочих часов (4, 6, 8, 9 часов и т.д.)

3. **Aliases - объединение нескольких ID**
   - Учёт нескольких ID как одного человека
   - Полезно при смене карт доступа или изменении ID в системе
   - Автоматическое разрешение главного ID

## 📋 Структура person_mapping.json

```json
{
  "person_mappings": {
    "ID_сотрудника": {
      "display_name": "Отображаемое Имя",
      "original_names": ["Вариант 1", "Вариант 2"],
```

## 💻 Примеры использования

### Базовая загрузка с маппингом:

```python
from services import DataLoader, PersonMapper

# Инициализация маппера
mapper = PersonMapper('person_mapping.json')

# Загрузка логов с применением маппинга
df = DataLoader.load_logs(
    'LogsCam/vhod.ndjson',
    file_type='ndjson',
    person_mapper=mapper
)

# В DataFrame будут поля:
# - name: person_id (главный ID с учётом aliases)
# - display_name: отображаемое имя
```

### Добавление нового сотрудника:

```python
mapper = PersonMapper('person_mapping.json')

mapper.add_person(
    person_id='99',
    display_name='Иван Иванов',
    original_names=['Иван Иванов', 'И. Иванов'],
    workdays=['Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday'],
    start_time='09:00',
    end_time='18:00',
    work_hours=9
)

mapper.save_mappings()
```

### Объединение нескольких ID:

```python
mapper = PersonMapper('person_mapping.json')

# Все события с ID 54, old_54, new_54 будут учитываться как один человек
mapper.add_alias(
    main_id='54',
    alias_ids=['old_54', 'new_54']
)

mapper.save_mappings()
```

- Aliases работают только на уровне ID, не имён

- Максимальная вложенность aliases: 1 уровень (нет цепочек)

## 👥 Для разработчиков

### API PersonMapper:

```python
class PersonMapper:
    def resolve_person_id(self, employee_id, name=None) -> str
    def get_display_name(self, person_id) -> str
    def get_schedule(self, person_id) -> Dict
    def process_event(self, event) -> Tuple[str, str]
    def add_person(...) -> bool
    def add_alias(main_id, alias_ids) -> bool
    def save_mappings() -> bool
    def convert_to_prefs_format() -> Dict
```

### Интеграция в существующий код:

```python
# До:
df = DataLoader.load_logs('logs.ndjson', file_type='ndjson')

# После:
mapper = PersonMapper('person_mapping.json')
df = DataLoader.load_logs('logs.ndjson', file_type='ndjson', 
                         person_mapper=mapper)
```
